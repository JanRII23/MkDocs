# :cooking: System Design: Live Comments

Designing a system like __live comment system__ enables users to post and receive comments in near real time while watching a live stream. The architecture typically begins with clients connecting to a fleet of stateless comment servers through a load balancer. When a user submits a comment, the request is validated, persisted to a durable data store for reliability, and then published to a distributed messaging layer. Because popular live streams can generate thousands of comments per second, the system must be horizontally scalable, fault tolerant, and capable of delivering updates with minimal latency to viewers across multiple regions..

<!-- more -->

A distributed pub/sub platform such as Apache Kafka or Redis Pub/Sub is commonly used to fan out comments to connected viewers. Each live stream can be mapped to one or more topics or channels, allowing comment servers to subscribe only to the streams they serve. When a comment is published, subscribed servers receive the event and immediately push it to clients using WebSockets or Server-Sent Events. This decouples comment producers from consumers, improves scalability, and allows the system to absorb traffic spikes through message buffering and partitioning. Additional services such as moderation, spam detection, analytics, and notification systems can independently consume the same comment stream without impacting the core user experience.

Apache ZooKeeper provides coordination and service discovery for the distributed infrastructure. It can maintain metadata about active comment servers, messaging brokers, and stream ownership, allowing components to dynamically discover one another and recover from failures. ZooKeeper also supports leader election, configuration management, and distributed locking, which help prevent duplicate processing and ensure consistent cluster state during server outages or scaling events. Together, ZooKeeper and the pub/sub layer create a resilient architecture where comment traffic can be distributed efficiently across many servers while maintaining high availability and low-latency delivery for millions of concurrent viewers.
