# :cooking: System Design: Dropbox

Designing a system like __Dropbox__ is about building a scalable, reliable file storage and synchronization platform that keeps data consistent across devices. The core challenge isn’t just storing files—it’s efficiently syncing changes, handling large-scale storage, and minimizing bandwidth while maintaining durability and availability.

<!-- more -->

A Dropbox-like system is a distributed platform designed to store files reliably and keep them synchronized across multiple devices. At a high level, it separates metadata (file structure, ownership, versions) from actual file storage (chunked blobs in object storage like Amazon S3) to scale efficiently. When users upload files, they are split into chunks, hashed, and only modified pieces are transferred (delta sync) to reduce bandwidth. The system uses deduplication to avoid storing duplicate data, strong consistency for metadata to ensure correctness, and eventual consistency for file propagation across devices. It also handles conflicts by creating separate versions and relies on caching, replication, and horizontal scaling to maintain low latency, durability, and high availability under heavy usage.

Primary points:

- Chunk-based storage with hashing and deduplication
- Delta sync for efficient uploads/downloads
- Separation of metadata and blob storage
- Strong consistency (metadata) vs eventual consistency (files)
- Conflict resolution via versioning
- Horizontal scaling, caching, and replication for performance and reliability
