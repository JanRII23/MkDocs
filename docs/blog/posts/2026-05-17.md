# :cooking: System Design: WhatsApp

Designing a system like __WhatsApp__ is a large-scale distributed messaging platform designed to support billions of users with low latency, high availability, and real-time communication. Its architecture is built around persistent TCP or WebSocket connections that allow clients to maintain continuous communication with chat servers rather than relying on repeated HTTP requests. Traffic is routed through Layer 4 and Layer 7 load balancers to efficiently distribute connections and maintain session consistency. The chat servers themselves handle responsibilities such as maintaining active user connections, tracking online presence, routing messages, processing acknowledgements, and coordinating message retries for offline users.

<!-- more -->

When a user sends a message, the client forwards it to a chat server, which authenticates the request and publishes the event into a distributed messaging pipeline. Systems such as Apache Kafka or Redis Pub/Sub are commonly used to asynchronously propagate messages between services and recipient servers. Redis Pub/Sub is particularly useful for low-latency events like typing indicators and presence updates, while durable systems like Kafka provide reliable event persistence and replay capabilities. If a recipient is online, the message is delivered immediately; otherwise, it is temporarily stored and paired with a mobile push notification until the user reconnects.

To scale efficiently, WhatsApp-like systems separate workloads across specialized storage layers. Relational databases such as PostgreSQL or MySQL manage structured account and authentication data, while distributed NoSQL databases such as Apache Cassandra store massive volumes of chat history with high write throughput. Additional infrastructure such as Apache ZooKeeper may be used for service discovery and cluster coordination. The system is horizontally scaled across many servers and regions to ensure fault tolerance, reliable message delivery, and consistent performance under massive global traffic loads.
