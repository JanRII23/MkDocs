# :cooking: System Design: Top-K

Identifying the most popular items from millions of events is a common requirement in modern distributed systems. Whether tracking trending YouTube videos, top search queries, popular hashtags, or most-viewed products, organizations need a scalable way to continuously determine the Top-K items from massive streams of data. A Top-K system must efficiently ingest events, maintain accurate frequency counts, and provide near real-time rankings while handling high throughput, low latency, and distributed processing challenges. Designing such a system requires careful consideration of storage, stream processing, fault tolerance, and memory-efficient counting techniques.

<!-- more -->

At a high level, a Top-K system begins with users generating events that are ingested through load balancers and distributed messaging platforms such as Kafka. Stream processing frameworks like Flink or Spark Streaming consume these events and aggregate counts for each item. For smaller datasets, exact counts can be maintained using distributed hash maps or databases. However, at internet scale, storing every unique key becomes expensive, so probabilistic data structures such as Count-Min Sketch are often used to provide approximate frequency estimates while significantly reducing memory consumption. These processors continuously update counts and maintain candidate rankings as new events arrive.

To efficiently retrieve the most popular items, the system typically combines frequency counting with a Min-Heap that stores only the current Top-K candidates. As item counts increase, the heap is updated and lower-ranked entries are evicted. Periodic checkpointing and state snapshots ensure fault tolerance, allowing stream processors to recover without losing progress. Results can be cached in Redis and exposed through APIs to serve real-time leaderboards and trending content. This architecture enables organizations to process millions of events per second while maintaining accurate and scalable Top-K rankings with predictable resource usage.
